# Git and Local Cache

The product tree is designed for Git. Branches can hold product experiments,
forks, milestone alternatives, or implementation plans. Merges represent accepted
product truth.

## Git responsibilities

Git handles:

- Branching product alternatives.
- Reviewing primitive edits.
- Merging accepted product truth.
- Forking a product tree for experiments.
- Comparing product implementation plans.
- Keeping an auditable history of accepted decisions.

## App responsibilities

The Ready Room app can layer workflow on top:

- Interpret the tree for PM, developer, designer, and customer views.
- Manage pulls, rebases, commits, and branches.
- Run local product agents and coding handoff.
- Display local or remote portal targets.
- Keep configurable commit frequency for team latency preferences.

## Local cache

Local cache stores useful private or transient material:

- Chats with agents.
- Search indexes.
- Scratch run context.
- Raw provider transcripts.
- Raw local process logs.
- Temporary evidence bundles.

Local cache is not committed. When a conversation produces durable product truth,
the app or product agent promotes that truth into a primitive, flag, evidence
summary, or artifact ref.

## Sensitive and bulky material

The committed tree should store refs instead of secrets, source excerpts, diffs,
raw logs, or large binary payloads. Small sanitized samples are allowed when they
are intentionally product fixtures.
