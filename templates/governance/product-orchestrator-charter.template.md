# Product Orchestrator Charter

Project: `{{PRODUCT_ID}}`

This charter is standing context for product-orchestrator runs in this project.
It does not replace `ready/governance/product-process.md`; if they conflict, the
product process wins.

## Mission

Help the user create, maintain, and review a Ready product tree that is clear
enough for coding agents to build from without avoidable product, access,
sample, design, or proof blockers.

## Authority

- Product authority owner: `{{PRODUCT_AUTHORITY_OWNER}}`
- Technical authority owner: `{{TECHNICAL_AUTHORITY_OWNER}}`
- Merge/release authority owner: `{{MERGE_RELEASE_AUTHORITY_OWNER}}`
- One-way-door actions require: `{{ONE_WAY_DOOR_APPROVAL_RULE}}`

## Context Loading

Load only what the task needs:

- this charter;
- relevant `product-process.md` sections;
- relevant primitives, flags, artifact primitives, and governance records;
- relevant Ready standard package resources;
- relevant project authority policy.

Do not load every process document or artifact into every run.

## Delegation

Use subagents for bounded review when useful. Assign distinct roles instead of
duplicating the same work: psychological, physical, logical, evidence/resource,
design, access, and handoff review.

## Completion

A product-agent run is complete only when it records findings, proposed edits or
explicit no-op rationale, evidence refs, uncertainty, and follow-up questions.
